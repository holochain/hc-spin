### Miscellaneous Tasks

- Exclude generated CHANGELOG.md from Prettier by @ThetaSinner in [#83](https://github.com/holochain/hc-spin/pull/83)
  - The changelog is regenerated by git-cliff at release time and its output is not Prettier-formatted, which fails format:check during the release flow.
- *(deps)* Bump @holochain/hc-spin-rust-utils to 0.700.0-dev.3 by @ThetaSinner
- Update flake lock by @ThetaSinner
- *(deps)* Upgrade electron to 42 and refresh build/lint tooling by @ThetaSinner
  - Electron 29 -> 42 (+ @electron-toolkit/preload, utils) - electron-vite 2 -> 5, vite 5 -> 7 (electron-vite 5 does not yet   support vite 8), electron-builder 24 -> 26 - eslint 8 -> 10 with migration from .eslintrc to flat config   (eslint.config.mjs); @electron-toolkit eslint configs 1 -> 3.   Original lint surface preserved (explicit-function-return-type off,   require/any allowed, caughtErrors none). - typescript 5 -> 6, @types/node 18 -> 22 (match flake/Electron node),   prettier, simple-import-sort, rimraf, commander, @msgpack/msgpack,   nanoid, get-port, bufferutil, utf-8-validate bumped - @holochain/client -> 0.20.5, hc-spin-rust-utils -> 0.700.0-dev.2   (kept within the Holochain 0.7 line) - bundle the now ESM-only electron-context-menu via externalizeDepsPlugin - make the cli.js electron-version warning track the build-time version

### CI

- Pin Node to 24 in CI and the nix dev shell by @ThetaSinner
  - The upgraded toolchain raises the Node floor (eslint 10 requires ^20.19 || ^22.13 || >=24; electron 42 wants >=22.12). Pin CI via setup-node and bump the flake dev shell from nodejs_22 to nodejs_24 so the engine requirements are satisfied deterministically.

## 2026-02-13: v0.700.0-dev.1

- Holochain is now run with the iroh transport. The argument `--signaling-url` has been replaced with `--relay-url`.

## 2026-01-13: v0.700.0-dev.0

### Changed

- Updated to holochain 0.7.0-dev.7

## 2025-11-20: v0.600.0

### Added

- A new argument `--force-admin-ports` takes a comma-separated list of port numbers, to force generated conductors to use specific admin ports. [#50](https://github.com/holochain/hc-spin/pull/50)

## 2025-11-06: v0.600.0-rc.0

### Added

- A new argument `--target-arc-factor` to support overriding the conductor configuration `network.target_arc_factor` in all launched conductors.

### Fixed

- Fixed development environment setup and developer documentation outlining how to use it.

### Changed

- Check linting in CI workflow
